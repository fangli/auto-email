File Templates

Place these files in the same directory as your CSV file. Each file is a template that can use {{ column_name }} placeholders, which get replaced with values from each CSV row.

CSV file — Your recipient data. The first row is headers; each subsequent row is one email. A _status column is added automatically to track sent/skipped/pending.

name,email,company,invoice
Alice,alice@example.com,Acme Inc,INV-001
Bob,bob@example.com,Globex,INV-002

email_recipient_template.txt — Who to send to. Supports comma-separated addresses.

{{ email }}

email_subject_template.txt — The email subject line.

Invoice {{ invoice }} for {{ company }}

email_body_template.txt — The email body. Can be plain text or HTML.

Hi {{ name }},

Please find attached invoice {{ invoice }} for {{ company }}.

Best regards,
Finance Team

email_attachment_template.txt (optional) — Path to attachment file(s). Can resolve to empty (no attachment), a single path, or comma-separated multiple paths.

invoices/{{ invoice }}.pdf

Example with multiple attachments:

invoices/{{ invoice }}.pdf, contracts/{{ company }}.docx

How it works: For each CSV row, the tool resolves all {{ }} placeholders in every template file using that row's column values. Emails are sent using the gws gmail +send command automatically. HTML bodies are detected and sent with the --html flag. You review each email in this portal before clicking Send.
